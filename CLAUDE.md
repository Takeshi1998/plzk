# PLZK.GG — プロジェクト概要

> このファイルはAIコーディングエージェント向けのコンテキスト。
> 現状は「LP + サービスUIモック」の静的HTML段階。まだ本番アプリのコードは無い。

## 1. 何を作っているか

CoD: Black Ops 1 / 2 のPS5・PS4移植(2026年7月、Treyarch/Iron Galaxy)に合わせて、
かつての日本のCoDコミュニティ文化（plzk / SR2on2 / CyAC縛りサーチ / クラン戦）を、
**レート・戦績・信用が資産として残る非公式Webサービス**として再構築する。

一言でいうと「**戦績が残るplzk（ランク付きの対戦相手募集掲示板）**」。
単なる募集掲示板ではなく、試合結果がElo・ランキング・プロフィールに蓄積されることが価値の核。

- サービス名: **PLZK.GG**（想定ドメイン plzk.gg）
- 運営: 個人開発
- 位置づけ: **非公式ファンコミュニティ**（Activision非提携）

## 2. ターゲット

- 27〜33歳、BO2時代（2012〜）に中高大生だった元サーチ勢
- 当時のクランは解散、フレンドは音信不通。腕には多少自信が残っている
- 社会人で夜2〜3時間のプレイ時間、可処分所得はある
- 二次ターゲット（BO6/BO7現役勢・配信者）はLPでは狙わない

## 3. コンセプト / 訴求

回顧（追悼）ではなく **運動（決起）**。主語は「俺ら」。過去形を使わない。

- 旗: 「plzkを再度盛り上げる」「5v5・SR2on2のランク化」
- ランクは主役ではなく **盛り上がりを一過性で終わらせないための装置** と位置づける
- Heroコピー: **「plzk、再稼働。」**
- トーン: 当時の界隈語彙をそのまま使う（plzk / 縛りサーチ / 混合 / ホスト / STANDOFF 等）。
  部外者に伝わらなくてよい。内輪性がターゲティングフィルタ兼・本物である信頼シグナル

## 4. サポートするゲームモード（両方ランク対応）

### SR2on2（SITE A / オレンジ）
- 2v2 のSearch & Destroy、CyAC準拠ルール
- 個人レート（Elo）。相方は固定でも都度でもOK

### 5v5 Search（SITE B / ブルー）
- チーム対チームの対戦相手探し
- **メンバー集めは外部（Discord/X）で完結済み**の前提。サイトには組み上がった5人チームを登録し、相手チームを募集・応募する
- チーム平均レートを表示。**「一人で参加」導線はサイトにはない**（ソロ復帰勢の受け皿はDiscordの混合イベント + SR2on2の相方募集）

> ⚠️ 重要: 5v5は「開催枠に一人ずつ参加する」ワイワイ開催モデルではない。
> SR2on2と同じ「チーム単位で相手を探して応募する」構造で統一されている。

## 5. 設計上の確定事項（実装時に踏襲すること）

### アカウント / 認証
- アカウント本体は **OAuth単位（Discord / X ログイン）**
- **PSN IDはコミュニケーション用の自己申告**。実在確認はしない
  （SonyはPSN公開APIを出しておらず、非公式APIは本番導線に組み込まない方針）
- なりすまし対策は事後（通報→手動確認）。事前必須のワンタイム認証は将来（Dフェーズ）

### レート反映ルール
- **レートが動く条件: 両陣営に登録ユーザーが最低1人ずつ**
- 自陣2人（SR2on2）が登録済み＝自分たちのレートがフルに動く
- 相手側が全員未登録なら「記録は残るがアンレート」
- 未登録の相方/相手は **ゲスト**（PSN IDは記録に残るがレート計算から除外）
- ゲストは後から登録すれば自分の試合をレートに反映できる（登録の動機づけ）
- レート方式は Elo。5v5はチーム平均レート同士で期待勝率→勝者5人一律+/敗者5人一律−
- **計算式は差し替え可能に設計**（初日から「誰と誰が組んだか」等のデータは全記録）

### Elo と 信用スコアの分離
- Elo = 強さ、信用スコア（承認率・No Show率・通報率）= 行動、の二軸で **分ける**
- 虚偽報告やNo ShowのペナルティをEloに混ぜない（ランクの意味が壊れるため）
- MVPでは信用スコアの **表示は不要、記録だけ初日から**

### マッチ後のやり取り
- **サイト内チャットは作らない**。実際のロビー合流はPSN/Discordで完結
- マッチ成立時に「マッチページ」（静的な情報ページ）を生成し、両陣営のPSN ID・ホスト指定・
  適用ルール・結果報告/承認/異議ボタンだけを提供
- サイトの責務は **構造化されたイベント（成立→報告→承認→異議）の記録**。会話はオフサイトに逃がす
- No Show対策の「チェックイン」ボタンはBフェーズで検討

## 6. MVPスコープ

### A. 必須
LP+X投稿テンプレ / Discordログイン+PSN ID登録 / SR2on2のMatch Finder（手動募集一覧）/
結果報告+相手承認 / Elo（仮レート表示）+シーズンランキング / プロフィール・戦績 / Discordサーバー

### B. 入れてもよい
異議時のみのスクショ提出（全試合必須提出はしない）/ Discord通知 / 相方募集 / 簡易No Show処理

### C. 作らない（初期）
自動マッチング / Ranked Queue / Accept確認 / マッチルーム自動化 / Glicko /
電話番号認証 / 動画提出 / BAN自動化 / 大会機能の内製（初期はTonamel+Discord人力）/ 課金・広告

### D. 将来（試合数が増えてから）
自動マッチング / 信用スコア表示 / 虚偽報告ペナルティ自動化 / 同一相手Elo減衰 /
CyAC風の5v5厳格ルール拡張 / 英語対応 / 課金 / チーム固定デュオ保存

### 判断の要点
- 自動マッチングは不要。人口が薄い初期は待ち時間で死ぬ。手動Match Finderで始める
- レートは必須。Admin裁定は「機能」でなく運営本人がDiscordで裁く
- 国内のみ（日本固有文化が参入障壁そのもの）

## 7. 戦略メモ

- 移植の人口は発売後1〜3ヶ月で急減する前提。「持続」ではなく **「3ヶ月の祭りを取り切る」** 設計
- スケジュール感: 7月中にLP公開 → 8月頭にMVP → 8月にシーズン1開始。この速度が出せないならやらない
- 収益は月数万円レジームが現実。スタートアップにはならないが、個人の資産としては成立
- 事前登録フォームは作らず **CTAはDiscord参加一本**。人数がそのまま社会的証明＆Go/No-Go判断材料
- マイルストーン公開でRT動機を作る（300人=開幕杯確定 / 500人=混合週2 / 1000人=スポンサー）

## 8. 法務・ブランディングの制約（コード・コピー生成時に必ず守る）

- **CoDのロゴ・スクリーンショット・キーアート・フォント模倣を一切使わない**（著作権）
  - モック画面は自作UIのみ。AI生成画像も兵士/銃/CoDロゴを出さない
- サービス名やUIに **「CyAC」の名称を使わない**（他者ブランド）。「CyAC準拠ルール」の記述に留める
- フッター等に非公式表記を必須で入れる:
  「本サービスはActivision Publishing, Inc.および関連会社とは一切関係のない、非公式のファンコミュニティです。Call of Duty®はActivision Publishing, Inc.の商標です。」
- 大会: **参加費を賞金原資にプールするのは賭博罪リスク → 絶対にやらない**。
  賞金を出すならスポンサー原資。景表法（自社サービスの取引に付随する景品の上限）にも注意
- 別途運営しているApexサイト「エペタム」とは **ブランドを分離**（相互宣伝しない）。
  ただしFAQ等で運営者の実績としての開示はOK（個人運営の継続性への信頼になる）
